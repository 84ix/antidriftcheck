ZMK TrackPoint OS Filter Analyzer

ChromeでZMKのUSBシリアルログと、OS通過後のマウス移動イベントを同時取得し、OSが微小HID入力をどう扱っているか推定します。

0. 使い方

このページは、ZMKログ上の Mouse movement set to X/Y と、ブラウザが受け取った mousemove movementX/Y を時刻合わせして比較し、antidrift / jitter-no-move の効き具合を確認します。

  1. ChromeまたはEdgeで開き、USBシリアル接続でZMKログを取得します。貼り付け解析でも使えます。
  2. Pointer Lock開始を押し、TrackPointに触れず静止状態のドリフトを数十秒記録します。操作後の残留も見たい場合は、軽く動かしてから手を離します。
  3. 比較解析するを押し、ドリフト抑制率、可視残留、方向の偏りを確認します。
  4. Codex解析用JSON保存で、ログ・設定・解析サマリを含むJSONを保存します。このJSONはそのままCodexに投げて、追加解析やパラメータ提案に使う前提です。

ZMKが 1/0 や 2/0 を出しているのにブラウザ側が動かない場合は、OS/ブラウザ入力層、またはZMK側antidriftで微小移動が抑制された候補です。
